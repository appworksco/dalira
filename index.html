Introduction

Dalira is a PHP framework characterized by its lightweight nature, offering elegant syntax and rapid runtime for development tasks. The template is open-source, so you can modify it yourself.

Usage

Using the template is very simple and straightforward. You can download the latest version here. There are some requirements you should have first:

  • XAMPP
  • Composer

Once you unzip the file, you'll observe the file structure arranged in this manner.

dalira-template/
├── app/                ← Application directory
│   ├── layout/         ← Directory for headers and footers
│   ├── modals/         ← Directory for modals
│   └── models/         ← Directory for CRUD operations
├── config/             ← Configuration directory
├── public/             ← Public directory
│   ├── css/            ← Directory for CSS files
│   ├── img/            ← Directory for images
│   └── js/             ← Directory for JS files
└── vendor/             ← Libraries installed by Composer

Starter Template

The following code imports the header.php and footer.php files from a specified directory using their absolute paths, ensuring their inclusion regardless of the current directory. You're then able to place your content between the header and footer includes.

                            
<?php include realpath(__DIR__ . '/app/layout/header.php') ?>

<h1>Hello world!</h1>

<?php include realpath(__DIR__ . '/app/layout/footer.php') ?>

Models

Models are frequently utilized to streamline the handling of database interactions, covering a wide range of tasks including data retrieval, insertion, updating, and deletion. Typically, these models are structured within the specified directory arrangement, commonly found at ./app/models/. Below, you'll find a demonstrative example illustrating the creation of a model designed specifically to execute CRUD (Create, Read, Update, Delete) operations.

                                
<?php

class ModelsFacade extends DBConnection
{

    function create($data)
    {
        $sql = $this->connect()->prepare("INSERT INTO dalira_table VALUES (?)");
        $sql->execute([$data]);
        return $sql;
    }

    function read()
    {
        $sql = $this->connect()->prepare("SELECT * FROM dalira_table");
        $sql->execute();
        return $sql;
    }

    function update($data, $dataId)
    {
        $sql = $this->connect()->prepare("UPDATE dalira_table SET data = '$data' WHERE id = '$dataId'");
        $sql->execute();
        return $sql;
    }

    function delete($dataId)
    {
        $sql = $this->connect()->prepare("DELETE FROM dalira_table WHERE id = ?");
        $sql->execute([$dataId]);
        return $sql;
    }

}

This PHP code defines a class named ModelsFacade, extending another class DBConnection, likely responsible for managing database connections. Within ModelsFacade, four methods are implemented:

  1. create($data): Inserts data into a table named dalira_table by executing an SQL INSERT statement with the provided data. It returns the executed SQL statement.
  2. read(): Retrieves all data from the dalira_table by executing an SQL SELECT statement. It returns the executed SQL statement.
  3. update($data, $dataId): Updates data in the dalira_table based on the provided $data and $dataId, using an SQL UPDATE statement. It returns the executed SQL statement. However, note that direct interpolation of variables into the SQL string could pose a risk for SQL injection attacks.
  4. delete($dataId): Deletes data from the dalira_table where the ID matches the provided $dataId, using an SQL DELETE statement. It returns the executed SQL statement.
